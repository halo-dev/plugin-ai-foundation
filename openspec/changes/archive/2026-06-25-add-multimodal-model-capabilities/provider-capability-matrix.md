# Provider Capability Matrix

This matrix is change-local implementation evidence for `add-multimodal-model-capabilities`.
It is not a long-lived user-facing provider capability document.

Rules for implementation:

- Do not enable language media input, image generation, URL input, or concrete media-type support from model-name heuristics alone.
- Prefer explicit remote model metadata when it exists.
- Use provider-owned documentation as fallback evidence.
- If evidence is inconclusive, leave capability unknown/unsupported by default and allow manual admin override.
- Do not assume an OpenAI-compatible aggregator supports a capability only because OpenAI supports it.

## Implementation Notes

- `openailike`, aggregators, and providers with incomplete docs must not get optimistic capabilities.
- Coarse `vision` can remain a display/filter feature, but runtime capability checks must use `capabilities.language.imageInput`, `inputMediaTypes`, and `inputSources`.
- File/PDF support is not equivalent to image support. Only providers with explicit document/file request shapes should get `language.fileInput=true`.
- Provider file-upload IDs are not the same as AI Foundation `DataContent`; enable only the source types AI Foundation can represent (`data` or provider-native `url`).
- For generated image URLs, keep provider URL output as `GeneratedFile.url`; AI Foundation must not download it.
